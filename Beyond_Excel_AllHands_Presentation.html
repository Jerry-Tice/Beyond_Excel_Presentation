Case Study — Jerry, 8 min

The 8D System: Training, Facilitation & Personal Tracker

Three connected builds for the MBC 8D program — one for the room, one for the person running it, one for the work participants do between sessions. None of it went through a designer or a dev shop.

~4 hrs
Tracker app build time, including the massive rework
2
Languages throughout, English & Spanish
  • The training deck: we've always built our own — no designer, usually PowerPoint. Fine in a room, but remote participants can only watch. We rebuilt it as an interactive HTML session: a live problem-statement builder with AI polish and AI grading, a click-to-reveal 8D arc walkthrough, built-in speaker notes.
  • The facilitator guide: the same shift, on the presenter side. Not a printed script — a navigable tool with talking points, timing, and built-in objection-handling for exactly the pushback you'd get running this live.
  • The tracker app: a real web app — login, project dashboard, all eight disciplines, fishbone and 5-Why worksheets, fully bilingual. The twist: an AI review button that scores each discipline against the actual 8D methodology and tells the participant exactly what's missing before they present.
  • At MBC: the training deck ran as group facilitation — the room worked through it together live. The tracker app was each participant's own copy, used solo between sessions to log progress and bring it back to the next one.
Open 8D Tracker App Open Session 1 Training Deck

Speaker Notes

"We've never handed training decks off to a designer — just PowerPoint, built in-house. That's fine in a room, but it falls apart the second training goes remote. So we rebuilt the session as an interactive HTML file instead — the room still worked through it together at MBC, but the capability for individual remote participation is already built."

"Even my own facilitator guide became software instead of a printed script — sidebar navigation, timing built in, objection-handling cards for the pushback you actually get in the room."

"The tracker app is the other half. Total build time, including a full rework partway through, was about four hours. A dev shop would have quoted weeks and real money for it. At MBC, each participant used their own copy between sessions to log progress and bring it back for review."

Land the meta point: this all-hands presentation was built the same way as the training deck. Say so directly — it lands better than any slide could.

Logistics: Have Session 1 open in one tab and the personal 8D Tracker open in another so you can flip live and show both — the group session tool and the individual between-session tool — side by side. In Session 1, type a sample problem statement into the builder and click "Polish with AI" so the room watches it get rewritten and graded live — you'll need an Anthropic API key entered in that slide beforehand. Then flip to the Tracker and click "AI Review" on a sample project.

Case Study — Jerry, 8 min

MW360 Connect: A CI Project That Became a Product

What started as standard CI work turned into a live, agentic pipeline running inside the client's operation — start to finish, with no manual re-keying anywhere in the middle.

1
Order In

Online quoter fires a webhook with a JSON order package — no manual entry at intake.

→
2
Estimator Review

A person reviews the order and approves it — the human checkpoint before anything touches the ERP.

→
3
RPA into ERP

Approval triggers RPA to enter the order into the ERP — the platform has no API, so this replaces manual keying entirely.

→
4
AutoCAD Assembly

The app selects the right template(s) from the AutoCAD library and populates order-specific details across every product ordered.

→
5
Engineering Review

Finished drawings route to engineering for final review — humans still sign off before anything ships.

Speaker Notes

"My engagement at Millwork 360 started as a standard CI project. It became something more. This is what's possible when you combine operations expertise with the ability to build."

"Walk the room through the five steps left to right — order comes in as a JSON package over a webhook, an estimator approves it, RPA keys it into the ERP because the platform has no API, the app builds the AutoCAD drawings from the order details, and engineering signs off before it ships. Two human checkpoints — estimator and engineering — everything in between is automated."

Keep it at the "what it does" level — the RPA-because-no-API point is worth pausing on, since it shows this isn't a green-field system, it's built to work with whatever the client already has.

Case Study — Jerry, 8 min

Synovate LineBeat Help Desk: Built Once, Deployed Everywhere

The other builds solved one engagement's problem. This one is a Synergetics product — built once to serve every Synovate LineBeat client we have.

  • The problem: floor staff running Synovate LineBeat had no self-serve way to get answers or log an issue — everything routed informally, with no record and no consistency.
  • What we built: an authenticated web app with an AI help agent — bilingual, cites the actual SOP section, and hard-redirects any screenshot request straight to the manual instead of guessing. Behind it, a structured ticket system with auto-generated ticket numbers, a "My Tickets" view, an admin dashboard, and a knowledge-gaps page that surfaces what the AI couldn't answer.
  • Built to scale: multi-tenant from day one — one instance, client-isolated data, supports roughly five clients before we'd need to reassess the platform. This is infrastructure, not a one-off.

Speaker Notes

"Every build so far solved a problem for one engagement. This one is different — it's a product, built once, meant to sit under every Synovate LineBeat client we ever bring on."

"It's live-piloting right now at Rustic Bakery. Sixteen users invited, the core deliverables are essentially done, and we're finishing the end-to-end test before we call it complete. I'm not going to tell you it's finished — it's an honest pilot, and that's the point. This is what 'built once, reused everywhere' actually looks like in progress, not in a pitch deck."

Be straight about status if asked — D-01/D-02 are near complete, D-03/D-04 and the full supervised test are still pending. Don't oversell a live pilot as a finished product.
